WorkBrain: a living knowledge layer for how we work

An LLM-maintained work wiki that stores structure in a graph database and detail in Markdown — so answers are cheaper, more accurate, and always current. Same proven architecture as our personal Second Brain, retuned for the workplace.

Why it's different vs. RAG & plain wikis
The idea

A wiki the AI writes, organizes, and keeps tidy — for you

WorkBrain builds on the “LLM-maintained wiki” concept: knowledge lives as plain Markdown that a model reads and writes. We extend that with an explicit knowledge graph in a lightweight SQLite database. The graph holds the map of what we know — the people, projects, decisions, and concepts and how they connect. The Markdown holds the territory — the full detail behind each item. The AI keeps both in sync as work happens.

How it's built

Two layers, each doing what it's best at

SQLite graph database · the index

Nodes & relationships

One small row per item: a title, a one-line summary, its type, and typed links to related items. This is the searchable, traversable skeleton of everything we know.

  • Tiny rows → fast keyword and relationship search
  • Typed edges (“part of”, “decided in”, “owned by”) enable real traversal
  • Rebuildable index — never the source of truth
Markdown files · the detail

The full story per node

Each node points to a Markdown file holding the complete content — notes, context, history, links. The AI opens only the few files an answer actually needs.

  • Human-readable, portable, versioned — the source of truth
  • Detail is fetched on demand, not stuffed into every prompt
  • Lives in OneDrive — no extra storage, no new system to run
The core advantage

Why graph + Markdown beats keyword-only retrieval

Standard RAG (retrieval-augmented generation) chops documents into chunks and matches your question against them. If the keywords don't line up, the search comes up empty — even when the right answer exists under different wording. WorkBrain captures meaning and relationships, so it finds the right node even when the words differ, then follows the graph to pull in everything connected to it.

🧭

Meaning, not just matching

Items are organized by concept and connection. A question about “the onboarding revamp” finds the project node even if the doc calls it “new-hire flow v2”.

🔗

Follow the connections

One hit expands along typed edges — the project, its owner, the decisions behind it, the meetings it came from. RAG returns isolated chunks with no map between them.

⚡

Fewer tokens, lower cost

The model searches a compact index of summaries and opens only the handful of files it truly needs — instead of paying to stuff large chunks into every prompt.

WorkBrain, specifically

WorkBrain is the work-side counterpart to our personal Second Brain. The design is identical — graph index plus Markdown detail — but it's delivered entirely inside the tools we already use, with no separate web app to build, host, or secure.

Delivered as Cowork skills

WorkBrain runs as skills inside Cowork — no new interface for anyone to learn. People interact with it where they already work.
