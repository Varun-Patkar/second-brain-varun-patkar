## LMStudioClaw

Windows desktop agent runtime powered by local LM Studio models.

**Core idea:** Resident system-tray controller + local web UI. No model loaded at idle — loads model on session/automation start, runs agent, then unloads. Fills gap between always-on agents (OpenClaw) and nothing.

**Stack:** Python (64.1%), JavaScript (13.6%), PowerShell (13.1%), Shell, CSS, HTML. FastAPI backend, web frontend.

**Key features:**
- Skills, custom tools, MCP servers (native HTTP, not VS Code dependent)
- Consent-bounded filesystem
- Call-based: model only runs when needed — good for gaming PCs
- Inspired by Microsoft Cowork but native to LM Studio

**GitHub:** Varun-Patkar/LMStudioClaw (Public, main branch, 30 commits, 3 branches)

**Why built:** OpenClaw always-on burns GPU. Varun games frequently — wanted on-demand agent that also runs automations when PC is on, not burning resources 24/7.
